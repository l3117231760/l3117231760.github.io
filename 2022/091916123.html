C语言中数据的存储

C语言中的数据类型有很多：整形、浮点型、自定义类型，指针类型、void型。

       在计算机中数据以二进制的方式进行存储，C语言中定义变量就是在开辟空间，这个空间就像一个模子，放进去的时候不管他是是什么，出来的时候就是模子的形状，定义变量也是一样，不管他是怎么样的数据，他都会把数据放到你开辟的空间中去，当“边角料”过多就会发生截断，把多余的部分进行舍弃，C是保存位于低位的数据或者说是底权值的数据，只有在进行使用的时候才会关注它本身的性质进行解释而已。

整形数据的存储

char

这里不考虑大小端的问题，因为存进去的时候需要遵循大小端的规则，在取出来的时候同样需要遵循大小端的规则！！！

  • C语言中数据的存储为补码进行存储的，在数据的存储过程中其中第一位为符号位，其中0表示这个数为正，1表示这个数为负数，其中正数的原码、反码、补码是相同的，即十进制转化为二进制是什么样子就是什么样子，拿int来进行举例：
10
0000 0000 0000 0000 0000 0000 0000 1010 //原码
0000 0000 0000 0000 0000 0000 0000 1010 //反码
0000 0000 0000 0000 0000 0000 0000 1010 //补码
38=32+4+2
0000 0000 0000 0000 0000 0000 0010 0110//原码
0000 0000 0000 0000 0000 0000 0010 0110//反码
0000 0000 0000 0000 0000 0000 0010 0110//补码

  • C语言中负数的存储，原码、反码、补码便不再相同了拿int来进行举例：

    原码->补码

​ 反码 = 原码按位取反但是符号位不变

​ 补码 = 反码 + 1

​ 补码->原码

  1. 先求反码 反码 = 补码 - 1（日常使用）

    再求原码 原码=反码按位取反符号不变

  2. 再按 原码->补码的规则来一遍（计算机内部使用）

    先符号位不变按位取反然后加一

其中 负数的原码 + 补码 = 0；

浮点数的存储

float
double

浮点数的存储是依据国际标准IEEE（电气与电子工程协会）754标准，任何一个二进制浮点数都可以表示成下面的形式

  • (-1)^S x M x 2^E
  • (-1)^S表示符号位
  • M表示有效数字 大于等于1 小于2
  • 2^E表示指数位 ， float中 ， E为无符号数因为没有负数，所以把127当成0的分界点，需要把求得的数加上127

下面有两条规定

  1. 当E 为00000000的时候就把它当做接近于0 2的1-127次方 M前的一可以省略变为0. …
  2. 当E 为全1的时候可以理解为正负无穷取决于符号位

其中符号位在浮点数中用一位来进行表示，其他的M和E各不相同

#include<stdio.h>
int main()
{
    int a = 9;
    float *pi=(float*)&a;
    printf("%d\n",a);
    printf("%lf\n",*pi);
    *pi=9.0;
    printf("%d\n",a);
    printf("%lf\n",*pi);
    return 0;
};

浮点误差的解决方法

浮点数会有误差，就和我们十进制中的1/3一样是无限小数，可是存储的类型的内存是有限的，会产生误差，因为这种误差的存在当我们在进行判断的时候需要用其他的方式来。

可以通过引入

#include<stdio.h>
#include<float.h>
#include<math.h>
int main()
{
    double c=10.0;
    double a=0.9;
    if(c-a*10==0.1)
    {
        printf("bingo");
    };
    if(fabs(c-a*10)<FLT_DIG)
    {
        printf("xin bingo");
    };
    return 0;
};

输出

xin binggo
